# WIREFRAME_TEXTUAL_WEB_BASE

## Objetivo del wireframe

Este wireframe textual define una arquitectura base reusable para construir sitios web multipágina con navegación clara, lógica de conversión y módulos repetibles. Puede usarse como blueprint para diseño, desarrollo, copywriting o dirección estratégica.

## Página: Inicio

### Bloque 1. Hero principal

Objetivo:
Presentar propuesta de valor, orientar al usuario y abrir la ruta de conversión.

Contenido:

- etiqueta contextual
- titular principal
- subtítulo explicativo
- CTA principal
- CTA secundario
- lista corta de puntos de confianza

CTA:

- principal: solicitar diagnóstico / cotización / agenda
- secundario: ver soluciones / ver servicios / explorar catálogo

Reutilización:
Este bloque sirve para cualquier negocio si se cambia la promesa central y el CTA.

### Bloque 2. Problemas o necesidades resueltas

Objetivo:
Responder rápidamente qué resuelve la marca.

Contenido:

- título de sección
- 3 cards con problema, solución o beneficio

CTA:

- implícito hacia soluciones o contacto

Reutilización:
Puede convertirse en beneficios, dolores del cliente, líneas de servicio o ventajas competitivas.

### Bloque 3. Sectores o aplicaciones

Objetivo:
Mostrar amplitud de uso sin perder claridad.

Contenido:

- texto de contexto
- lista o grid de escenarios donde aplica la solución

CTA:

- explorar aplicaciones

Reutilización:
Funciona como bloque de industrias, públicos, usos o segmentos.

### Bloque 4. Proceso o metodología

Objetivo:
Explicar cómo trabaja la marca y reducir incertidumbre.

Contenido:

- título de sección
- secuencia de 3 pasos

CTA:

- hablar con el equipo

Reutilización:
Se adapta a servicios, productos complejos, procesos comerciales, onboarding o reservas.

### Bloque 5. Banner CTA final

Objetivo:
Cerrar la página con una acción concreta.

Contenido:

- titular corto
- CTA principal
- CTA secundario

CTA:

- agendar conversación
- ver casos / ver servicios

Reutilización:
Debe aparecer al final de páginas de alto interés.

## Página: Soluciones / Servicios / Productos

### Bloque 1. Hero interno

Objetivo:
Explicar el propósito de la página y ordenar la lectura.

CTA:

- implícito hacia secciones inferiores

### Bloque 2. Stack de soluciones

Objetivo:
Presentar oferta de forma modular.

Contenido:

- nombre de solución
- descripción breve
- CTA por card

CTA:

- solicitar evaluación
- ver aplicación
- pedir información

Reutilización:
Puede ser catálogo de servicios, paquetes, programas, productos o unidades de negocio.

### Bloque 3. Beneficios o criterios de decisión

Objetivo:
Ayudar a elegir y reforzar valor.

Contenido:

- grid de 4 puntos de valor

CTA:

- avanzar a contacto

Reutilización:
Se puede adaptar a ventajas, diferenciales, garantías o factores de decisión.

### Bloque 4. FAQ

Objetivo:
Reducir objeciones.

Contenido:

- 3 a 6 preguntas frecuentes

CTA:

- resolver caso con el equipo

Reutilización:
Apto para cualquier página de oferta.

## Página: Aplicaciones / Casos de uso / Sectores

### Bloque 1. Hero interno

Objetivo:
Conectar solución con contexto del usuario.

CTA:

- seguir explorando casos

### Bloque 2. Grid de contextos

Objetivo:
Ayudar a que el visitante se identifique.

Contenido:

- cards por tipo de caso
- necesidad resumida
- CTA contextual

CTA:

- consultar caso similar
- evaluar mi proyecto

Reutilización:
Puede convertirse en industrias, buyer personas, portafolio, colecciones o escenarios.

### Bloque 3. Patrón contexto-necesidad-respuesta

Objetivo:
Explicar cómo se traduce la oferta a cada caso.

Contenido:

- contexto
- necesidad
- respuesta

CTA:

- solicitar diagnóstico

Reutilización:
Muy útil en negocios consultivos y soluciones complejas.

## Página: Nosotros / Marca / Método

### Bloque 1. Hero interno

Objetivo:
Posicionar visión y tono de la marca.

CTA:

- conocer más

### Bloque 2. Narrativa de marca

Objetivo:
Construir credibilidad.

Contenido:

- bloque de visión
- bloque de diferencial

CTA:

- conversar con la marca

Reutilización:
Puede convertirse en historia, manifiesto, equipo, filosofía o método.

### Bloque 3. Principios

Objetivo:
Sintetizar la identidad en puntos claros.

Contenido:

- grid de 3 principios o valores

CTA:

- ir a contacto

Reutilización:
Puede convertirse en valores, promesas, garantías o pilares.

## Página: Contacto / Conversión

### Bloque 1. Hero interno

Objetivo:
Preparar al usuario para la conversión.

CTA:

- completar formulario

### Bloque 2. Formulario principal

Objetivo:
Capturar leads con suficiente contexto.

Contenido:

- nombre
- correo
- tipo de proyecto
- mensaje

CTA:

- enviar solicitud

Reutilización:
Puede adaptarse a agenda, reserva, cotización, demo o compra consultiva.

### Bloque 3. Sidebar de apoyo

Objetivo:
Ofrecer canales alternos y mejorar calidad del lead.

Contenido:

- canal directo
- ubicación
- guía de qué información compartir

CTA:

- escribir por canal alterno

Reutilización:
Sirve para soporte, ventas, alianzas o postventa.

### Soluciones

2. Stack de soluciones
3. Beneficios o criterios de decisión
4. FAQ

### Aplicaciones

1. Hero interno
2. Grid de contextos
3. Patrón contexto-necesidad-respuesta

### Nosotros

1. Hero interno
2. Narrativa de marca
3. Principios

### Contacto

1. Hero interno
2. Formulario principal
3. Sidebar de apoyo

## Variantes posibles por bloque

- hero con imagen, video, mockup o composición abstracta
- cards en 2, 3 o 4 columnas
- proceso en línea de tiempo o checklist
- FAQ desplegable o estático
- CTA final con un solo botón o doble acción
- formulario simple o enriquecido según nivel de venta

## Notas de reutilización

- mantener siempre una relación clara entre bloque y objetivo
- no añadir secciones decorativas sin función estratégica
- reutilizar módulos antes de crear nuevos
- adaptar el CTA al nivel de intención del usuario
- usar este wireframe como base, no como límite rígido
