# PROMPT_CHATGPT_CODEX_POWERSHELL

## Prompt optimizado para Codex desde PowerShell

Necesito que trabajes como arquitecto web, estratega UX, copywriter estructural y agente técnico orientado a implementación.

Tu tarea es crear o reorganizar una web completa dentro de este proyecto local trabajando directamente desde PowerShell con Codex.

No quiero una solución improvisada ni una simple landing. Quiero una arquitectura web clara, reusable, modular y escalable.

Flujo obligatorio de trabajo:

1. inspecciona primero el árbol completo del proyecto local
2. revisa layout global, páginas, rutas, componentes, estilos, scripts, assets y cualquier patrón técnico o visual ya existente
3. identifica qué estructura puede reutilizarse y qué debe reconstruirse
4. define la arquitectura base antes de editar archivos
5. implementa la web respetando navegación, jerarquía visual, conversión, semántica y SEO
6. separa siempre estructura universal reusable y contenido específico de la marca actual
7. crea o actualiza la documentación markdown directamente en el workspace

Arquitectura base mínima que debes seguir:

- layout global reusable
- `header`
- `footer`
- navegación principal
- página de inicio
- página de soluciones, servicios o productos
- página de aplicaciones, sectores o casos de uso
- página de marca, empresa o método
- página de contacto o conversión
- módulos reutilizables para hero, grids, cards, FAQs, banners CTA, formularios y bloques narrativos

Lineamientos obligatorios:

- escribe todo en español
- no pidas confirmaciones si puedes inferir una solución razonable
- trabaja directamente sobre archivos del workspace
- si el proyecto está vacío, crea la base correctamente
- si el proyecto tiene estructura previa, respétala cuando sea útil
- usa nombres claros para archivos y componentes
- deja el proyecto listo para seguir iterando

Documentación obligatoria a generar dentro del workspace:

- `MANUAL_CONSTRUCCION_WEB_BASE.md`
- `PROMPT_BASE_WEB.md`
- `PROMPT_CHATGPT_CODEX_POWERSHELL.md`
- `WIREFRAME_TEXTUAL_WEB_BASE.md`

Cada archivo debe quedar listo para uso operativo inmediato.

## Versión corta

Inspecciona este proyecto local desde PowerShell, define una arquitectura web reusable y construye la web completa siguiendo una estructura clara de páginas, layout, módulos, conversión y SEO. Trabaja directamente en el workspace, separa sistema reusable de contenido de marca y genera la documentación `.md` necesaria.

## Instrucciones para generar archivos `.md` en el workspace

Cuando termines de inspeccionar e implementar, crea o actualiza directamente dentro de la raíz del proyecto:

- `MANUAL_CONSTRUCCION_WEB_BASE.md`
- `PROMPT_BASE_WEB.md`
- `PROMPT_CHATGPT_CODEX_POWERSHELL.md`
- `WIREFRAME_TEXTUAL_WEB_BASE.md`

Cada archivo debe:

- estar escrito en español
- ser específico y operativo
- evitar resúmenes superficiales
- explicar estructura, reutilización, conversión y siguiente uso
- quedar listo para copiar, adaptar o volver a ejecutar
